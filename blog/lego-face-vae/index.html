Building the training set

I threw together some quick scripts to pull images of Lego faces from two sources: Bricklink and Quartz.

The high-level, handy-wavy explanation of how a VAE works

An autoencoder (AE) has two main parts: an encoder and a decoder. The encoder takes input (in this case, images of Lego faces each represented as 64x64x3 matrices) and outputs a vector that represents that input. The decoder takes a vector as input and yields output in the form of the input of the encoder (again, in this case, images of Lego faces represented as 64x64x3 matrices).
